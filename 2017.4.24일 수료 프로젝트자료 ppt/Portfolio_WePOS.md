## 💡 기획 의도

### 컨셉

```
We = Manager + Customer
WePOS = We + POS
```

> **WePOS**는 기존 POS(Point of Sale) 시스템의 개념을 웹으로 확장하여,  
> **매장 관리자**와 **고객** 모두가 함께 사용할 수 있는 **웹 기반 통합 매장 관리 플랫폼**입니다.

### 핵심 가치

| 가치 | 설명 |
|------|------|
| 🏪 **매장 관리** | 주문, 결제, 테이블 관리를 웹으로 통합 |
| 👥 **고객 경험** | 매장 검색, 예약, 리뷰를 하나의 플랫폼에서 |
| 📊 **데이터 분석** | 매출 통계, 차트를 통한 경영 인사이트 |
| 🔐 **권한 관리** | 관리자/일반 사용자 역할 기반 접근 제어 |

## ⭐ 주요 기능

### Overview

```
┌───────────────────────────────────────────────┐
│                 WePOS 기능 구조                 │
├───────────┬───────────────┬───────────────────┤
│   회원     │   매장 (관리자) │   검색/커뮤니티   │
├───────────┼───────────────┼───────────────────┤
│ • 로그인   │ • 영업 관리    │ • 매장 검색       │
│ • PW 찾기  │ • 매장 관리    │ • 상품 검색       │
│   (이메일) │ • 매출 통계    │ • 매장 상세보기   │
│ • 권한별   │   (표/차트)    │ • 게시판         │
│   기능 분리│ • 주문/결제    │ • 예약 관리       │
└───────────┴───────────────┴───────────────────┘
```

### 1. 회원 관리

| 기능 | 설명 |
|------|------|
| **로그인** | 권한별 기능 구분 (관리자/일반 회원) |
| **비밀번호 찾기** | 이메일 인증 기반 비밀번호 복구 |
| **권한 관리** | 관리자 → 매장 관리 기능 접근, 일반 회원 → 검색/예약/리뷰 |
| **회원별 메뉴** | 로그인 사용자에 따라 동적 메뉴 표시 |

### 2. 매장 관리 (관리자 전용)

| 기능 | 설명 |
|------|------|
| **영업 관리** | 테이블 주문 현황, 매장 현황, 실시간 매장 영업 화면 |
| **매장 관리** | 매장 정보 수정, 상품 관리, 테이블 설정 |
| **주문/결제** | 테이블별 주문 등록 및 결제 처리 |
| **매출 통계** | 표 + 차트 기반 매출 분석 (일별/월별/상품별) |

### 3. 검색

| 기능 | 설명 |
|------|------|
| **매장 검색** | 매장명/지역/카테고리 기반 검색 |
| **상품 검색** | 상품명/카테고리 기반 검색 |
| **매장 상세** | 매장 상세 정보, 메뉴, 위치, 리뷰 확인 |

### 4. 예약 관리

| 기능 | 설명 |
|------|------|
| **예약 등록** | 일반 회원의 매장 예약 |
| **예약 조회** | 관리자/고객 각각의 예약 현황 확인 |

### 5. 게시판

| 기능 | 설명 |
|------|------|
| **공지사항** | 공지사항 게시판 |
| **일반 게시판** | 일반 사용자 커뮤니티 게시판 |
| **댓글** | 게시글 댓글 기능 |

## 🏅 성과 및 회고

### 기술적 성과

| 영역 | 내용 |
|------|------|
| **Spring MVC** | Controller → Service → DAO 3계층 아키텍처 구현 |
| **MyBatis** | Oracle DB와의 ORM 매핑, 복잡한 쿼리 처리 |
| **권한 관리** | 관리자/일반 회원 역할 기반 접근 제어 시스템 |
| **차트 시각화** | 매출 데이터의 차트 기반 시각화 |
| **실시간 현황** | 테이블 주문 현황, 매장 영업 상태 실시간 표시 |
| **DB 설계** | 26개 테이블의 관계형 데이터베이스 설계 |

### 핵심 학습 포인트

1. **대규모 DB 설계**: 26개 테이블 간의 관계 설정 및 데이터 정규화
2. **Spring MVC 아키텍처**: 실무 수준의 3계층 구조 프로젝트 경험
3. **POS 시스템 이해**: 주문 → 결제 → 통계의 비즈니스 프로세스 구현
4. **데이터 시각화**: 차트 라이브러리를 활용한 매출 데이터 분석
5. **팀 프로젝트 협업**: 4인 팀의 역할 분담과 Git 기반 협업

### 프로젝트 특징

> 🌟 **WePOS만의 차별점**
> - 단순 POS가 아닌 **매장 관리자 + 고객** 양면 플랫폼
> - 매출 통계의 **차트 시각화**로 경영 인사이트 제공
> - **권한 기반 기능 분리**로 안전하고 체계적인 시스템 구현
> - **26개 테이블**의 정교한 DB 설계
